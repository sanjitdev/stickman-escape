# Architecture

## Game Loop

```
Phaser.Game
 └─ update(time, delta)
     └─ LevelScene.update()
         ├─ InputSystem.update(dt)          → cached InputState snapshot
         ├─ Player.update(dt, inputState)   → state.update(dt, input)
         ├─ Enemy[].update(dt)              → patrol FSM
         └─ HUD.update(elapsed, coins, deaths)
```

## Scene Flow

```
Boot (generate placeholder textures)
 └─ Preload (load assets, create animations)
     └─ MainMenu
         ├─ LevelSelect → LevelScene ──► VictoryScene
         │                         ├──► GameOverScene (unused — deaths handled in-level)
         │                         └──► PauseScene (overlay)
         └─ Settings
```

## Player FSM

8 concrete states, all extend `PlayerState { enter(); update(dt, input); exit() }`.

`Player.switchTo(key)` maps string keys to state instances (factory pattern), eliminating circular imports:

```
idle ──► run (direction pressed)
idle ──► jump (jump buffered)
idle ──► dash (shift, cooldown clear)
idle ──► fall (not on ground)

run  ──► idle (no direction)
run  ──► jump / dash / fall

jump ──► fall (vy > 0)
jump ──► wallSlide (wall contact)
jump ──► dash

fall ──► idle/run (grounded)
fall ──► jump (coyote window OR double-jump)
fall ──► wallSlide / dash

wallSlide ──► wallJump (jump pressed)
wallSlide ──► fall (moved away from wall)
wallSlide ──► idle (grounded)

wallJump ──► fall (vy > 0)
wallJump ──► idle (grounded)

dash ──► idle/fall (timer expires)

* ──► dead (player.die() called from hazard overlap or enemy side-hit)
dead: scene handles respawn after 800ms delay
```

Physics constants live exclusively in `Config.ts PHYSICS` — no magic numbers in state files.

## Level System

Two files per level:

- `public/assets/tilemaps/level-N.json` — Tiled JSON (3 layers: Background, Platforms, Hazards).
  Generated by `scripts/generate-tilemaps.mjs`. Platforms layer uses `setCollisionBetween(1, 24)`.
  Hazards layer uses `setCollisionBetween(25, 36)`.

- `public/assets/data/levels/level-N.json` — Entity config: playerSpawn, exitPortal, coins[],
  enemies[], keys[], doors[], movingPlatforms[], fallingPlatforms[], checkpoints[].

`LevelScene.create()` loads both, builds the tilemap, spawns all entities, wires collision.

## Save System

localStorage key `stickman-escape-save`. Shape:

```ts
{
  unlockedLevels: number[],
  levelStats: Record<number, { bestTime, deaths, coinsCollected, totalCoins, stars }>,
  totalCoins: number,
  settings: { musicVolume, sfxVolume, fullscreen }
}
```

Singleton `saveSystem` exported from `SaveSystem.ts`. `updateLevelStats` merges (keeps best time, highest stars, most coins).

## Asset Pipeline

1. User downloads Kenney CC0 packs → places PNGs in `public/assets/sprites/` and OGGs in `public/assets/audio/`.
2. `BootScene` generates procedural placeholder textures for any key that isn't overwritten.
3. `PreloadScene` tries to load real assets; load errors are silently ignored (placeholders remain).
4. Animations defined once in `PreloadScene.createAnimations()` using spritesheet frame indices.

## Virtual Gamepad

Detected via `this.sys.game.device.input.touch`. Creates 4 on-screen buttons (←, →, ⬆, ⚡) that write directly to `InputSystem.touchLeft/Right/Jump/Dash`. InputSystem.update() merges these with keyboard state each frame.
